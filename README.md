# 📡 Pokédex Completa

Este é o meu primeiro projeto de destaque na jornada como desenvolvedor Front-End! É uma Pokédex funcional que consome dados reais da [PokeAPI](https://pokeapi.co/), permitindo explorar os todos os pokemons

## 🚀 Sobre o Projeto

O objetivo deste projeto foi aplicar conhecimentos de JavaScript assíncrono e manipulação dinâmica do DOM. A interface foi inspirada no design clássico de uma Pokébola, utilizando um layout responsivo e intuitivo.

### ✨ Funcionalidades
- **Listagem Dinâmica:** Carregamento automático dos Pokémon via API.
- **Busca Inteligente:** Filtre por nome ou número do ID em tempo real.
- **Filtro por Tipo:** Menu de seleção para visualizar tipagens específicas (Fogo, Água, Grama, etc.).
- **Paginação:** Navegação fluida entre páginas para melhor performance.
- **Cores Adaptativas:** O fundo de cada card muda dinamicamente (com gradiente diagonal) baseando-se no tipo do Pokémon.
- **Modal de Detalhes:** Clique em um Pokémon para ver sua Arte Oficial, Status de Combate, Habilidades e sua "História" (Flavor Text) extraída da API de Espécies.

## 🛠️ Tecnologias Utilizadas
- **HTML5:** Estruturação semântica.
- **CSS3:** Flexbox, CSS Grid, Gradientes e Animações (Keyframes).
- **JavaScript (ES6+):** - Consumo de API com `fetch` e `Async/Await`.
    - Manipulação de Arrays (`map`, `filter`, `find`, `slice`).
    - Event Listeners e manipulação de classes CSS.
- **Fonte:** Ubuntu (Google Fonts).

## 🧠 O que eu aprendi neste desafio?
Como iniciante, este projeto foi um divisor de águas. Os principais desafios superados foram:
1. **Promessas (Promises):** Entender como esperar os dados da API chegarem antes de renderizar a tela.
2. **Lógica de Paginação:** Calcular o índice de início e fim para mostrar apenas 12 itens por vez.
3. **UX/UI:** Criar um Modal que fosse acessível e que trouxesse informações de dois endpoints diferentes da API simultaneamente.
